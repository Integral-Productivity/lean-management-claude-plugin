# 5S (Workplace Organization)

## What It Is

5S is a five-step method for organizing the work environment to support flow, quality, safety, and visibility. It creates the orderly baseline that makes abnormalities detectable and standardized work followable.

## The Five Steps

1. **Sort (Seiri)** -- Remove everything not needed for current work. Red-tag items of uncertain need and quarantine for a defined period. If unused, discard. The goal: only what's needed is present.

2. **Set in Order (Seiton)** -- Arrange remaining items for optimal access. "A place for everything, everything in its place." Organize by frequency of use -- frequently used items within arm's reach, rarely used items stored further away. Use shadow boards, labels, and color-coding.

3. **Shine (Seiso)** -- Clean thoroughly. Cleaning is not housekeeping -- it is inspection. Cleaning reveals abnormalities: leaks, wear, damage, loose connections, configuration drift. Establish cleaning as a daily practice, not an occasional event.

4. **Standardize (Seiketsu)** -- Create visual standards for maintaining the first three S's. Photographs of the "correct" state, checklists, labeled storage locations, min/max markers. The standard makes "normal" visible so that "abnormal" stands out immediately.

5. **Sustain (Shitsuke)** -- Build habits and audit regularly. Sustain is where most 5S implementations fail. Without regular audits (part of Leader Standard Work) and cultural reinforcement, 5S degrades within weeks.

## Adaptation for Knowledge Work

5S applies to digital environments -- file organization, tool configuration, communication channels, code repositories, shared drives:

- **Sort:** Archive or delete unused files, channels, repositories, bookmarks, browser tabs, subscriptions.
- **Set in Order:** Consistent naming conventions, folder structures, tag taxonomies, channel organization.
- **Shine:** Review for broken links, outdated documentation, stale configurations, unused integrations. Digital "cleaning" reveals configuration drift.
- **Standardize:** Templates, style guides, repository structures, onboarding checklists that define "how we organize things here."
- **Sustain:** Periodic reviews (quarterly digital 5S), automated detection of drift (lint rules, naming convention checks).

## When to Use

- **Problem type alignment:** Types 1, 2. Foundational practice that prevents problems rather than solving specific ones.
- Often the first lean activity in a new environment -- creates the visual order that makes other lean tools possible.

## Common Mistakes

- **5S as an end in itself.** 5S is a foundation, not a destination. Organizations that do 5S but no other lean work have achieved tidiness, not improvement.
- **Sort without discipline.** Keeping items "just in case" defeats the purpose. If it hasn't been used in the defined period, it goes.
- **Skipping Sustain.** The first three S's are a project; Sustain is a practice. Without it, 5S is a one-time cleaning event.
- **Management-driven 5S without worker involvement.** Imposed organization doesn't stick and may actually hinder workers who had functional (if messy) systems.

## Relationship to Other Tools

- **visual-management:** 5S creates the visual order that makes visual management possible.
- **standardized-work:** An organized workplace supports standard work adherence.
- **jidoka:** Abnormalities are easier to detect in an ordered environment.
- **kaizen-events:** Many events begin with 5S as the first step.
- **leader-standard-work:** 5S audits are a standard LSW activity.
