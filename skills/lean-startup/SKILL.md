# Lean Startup & Innovation

## Scope

This reference covers lean thinking applied to product innovation, new ventures, and situations of high uncertainty (Smalley's Type 4 problems). Where traditional lean optimizes known processes, lean startup addresses the challenge of discovering what to build when both the problem and solution are uncertain.

## Core Concepts

### Build-Measure-Learn (BML) Loop

The fundamental activity cycle of lean startup. The loop runs in this order -- Build -> Measure -> Learn -- but is *planned* in reverse: start with what you want to learn, determine what to measure, then build the minimum necessary to measure it.

- **Learn:** What is the riskiest assumption? What do we need to learn to reduce uncertainty?
- **Measure:** What data would confirm or refute our assumption? What metrics matter?
- **Build:** What is the minimum product, prototype, or experiment that would generate that data?

The loop is designed for speed. The faster you cycle through BML, the faster you reduce uncertainty. Ries calls this "validated learning" -- learning that is backed by empirical data from real customer behavior, not opinions, surveys, or guesses.

### Minimum Viable Product (MVP)

The smallest thing you can build that enables a complete cycle through the BML loop. The MVP is not a crappy first version of the product -- it is a *learning vehicle* designed to test a specific hypothesis with minimum effort.

Types of MVPs:
- **Concierge MVP:** Manually deliver the service to a small number of customers. No technology needed. Tests whether customers value the outcome.
- **Wizard of Oz MVP:** Present an automated-looking interface but perform the work manually behind the scenes. Tests whether customers will engage with the solution.
- **Landing page MVP:** A webpage describing the value proposition with a call-to-action. Tests whether there's demand before building anything.
- **Single-feature MVP:** Build only the core feature, nothing else. Tests whether the core mechanism works.

**Common misunderstanding:** MVP does not mean "minimum effort product" or "launch with bugs." It means the minimum experiment that generates validated learning about the riskiest assumption.

### Innovation Accounting

Standard financial metrics (revenue, profit) don't work for early-stage innovation because the numbers are too small to be meaningful. Innovation accounting uses learning milestones instead:

1. **Establish the baseline:** What do current metrics look like? (Even if they're near zero.)
2. **Tune the engine:** Run experiments to improve key metrics toward the business model's requirements.
3. **Pivot or persevere:** Based on accumulated learning, decide whether to continue the current strategy (persevere) or fundamentally change direction (pivot).

### Pivot

A structured change in strategy without a change in vision. A pivot is not failure -- it is a completed learning cycle that revealed the current approach won't work. Types include: customer segment pivot, value capture pivot, channel pivot, technology pivot, zoom-in/zoom-out pivot.

The discipline is to pivot based on data (validated learning), not based on frustration or boredom. Premature pivots waste learning; delayed pivots waste time.

### Lean Canvas (Maurya)

A one-page business model template adapted from Osterwalder's Business Model Canvas for startup contexts. Nine blocks: Problem, Solution, Key Metrics, Unique Value Proposition, Unfair Advantage, Channels, Customer Segments, Cost Structure, Revenue Streams. Designed to be filled out in 20 minutes and revised frequently.

## Relationship to TPS Lean

Lean startup is a legitimate branch of lean thinking, not a separate framework. The connections:

- **Waste elimination:** In startup context, the greatest waste is building something nobody wants. BML eliminates this waste by validating demand before investing in production.
- **Pull:** MVP is pull-based -- build only what's pulled by a learning need, not pushed by a product roadmap.
- **Small batches:** Rapid BML cycles are small-batch experiments. Large-batch product development (build everything, launch, hope) is the startup equivalent of overproduction.
- **Scientific thinking:** BML is PDCA applied to product-market fit. Hypothesis -> experiment -> measure -> learn -> adjust.

## When This Tradition Applies

- **Problem type:** Type 4 (open-ended, no known solution). Also applicable to Type 3 breakthrough objectives where the path to the target condition is highly uncertain.
- **Context signals:** New product/service development, entering new markets, testing new business models, any situation where "what to build" is more uncertain than "how to build it."

## Common Mistakes

- **Building without hypotheses.** "Let's see what happens" is not lean startup -- it's unstructured experimentation. Every build should test a specific, falsifiable hypothesis.
- **Treating MVP as V1.** The MVP is not the first version of the product. It's the first experiment. The product may look nothing like the MVP.
- **Vanity metrics.** Measuring things that make you feel good (page views, downloads) rather than things that indicate value (retention, willingness to pay, repeat use).
- **Refusing to pivot.** Falling in love with the solution rather than the problem. Lean startup requires willingness to change direction based on evidence.
- **Applying lean startup to Type 2 problems.** If you know what the process should look like and it's just not performing, you don't need BML -- you need root cause analysis and standard improvement tools.
